# AI 作家委员会 前端设计规格

> 目标：工业级写作工具的前端体验，吸取 Cherry Studio / 酒馆 / Notion / Scrivener / Linear 的优点。
> 适用对象：会长（产品决策者）→ DS（实施开发）→ 后续迭代。

---

## 一、设计哲学

### 1.1 核心原则

1. **会长是灵魂**：界面服务于会长的创作流程，不是炫技
2. **信息密度可调**：极简模式聚焦写作，专业模式展示协作过程
3. **零打断创作**：长任务在后台进行，会长可同时干别的
4. **键盘优先**：所有高频操作有快捷键，鼠标用户也舒服
5. **状态可见**：任何时候会长知道"系统在做什么、还要多久"

### 1.2 风格定位

- **整体气质**：黑色背景 + 高饱和彩色强调（酒馆风）
- **节奏感**：紧凑但不拥挤，留白克制
- **细节**：微动效（200-300ms 过渡）、状态反馈（hover/active/loading）

## 三、整体布局

### 3.1 主结构

```
┌──────────────────────────────────────────────────────────────┐
│ TopBar (40px)                                                │
├──┬──────────────────────────────────┬───────────────────────┤
│  │                                  │                       │
│左│        主面板                     │   右侧抽屉           │
│侧│        （随模式切换）              │   （辩论流 / 设定）   │
│栏│                                  │   可隐藏              │
│  │                                  │                       │
│2 │                                  │                       │
│2 │                                  │                       │
│0 │                                  │                       │
│p │                                  │                       │
│x │                                  │                       │
│  │                                  │                       │
├──┴──────────────────────────────────┴───────────────────────┤
│ StatusBar (28px) - 进度/token/费用/快捷键提示                │
└──────────────────────────────────────────────────────────────┘
```

### 3.2 三大区域职责

| 区域 | 功能 | 可见性 |
|------|------|--------|
| **TopBar** | Logo、项目名、模式切换、主题切换、设置入口 | 常驻 |
| **LeftSidebar** | 项目列表、章节树、快速操作 | 可折叠（48px 图标态） |
| **MainPanel** | 主工作区，随模式切换布局 | 常驻 |
| **RightDrawer** | 辩论流、设定弹窗、角色卡 | 可隐藏 |
| **StatusBar** | 进度条、token 消耗、当前快捷键提示 | 常驻 |

### 3.3 拖拽与缩放

- 所有面板边界**可拖动调宽度**（最小宽 200px，主面板最小 400px）
- 拖动手柄：4px 宽不可见区域，hover 时显示蓝色高亮线
- 双击边界：回到默认宽度
- 状态持久化到 localStorage

## 四、双模式切换

### 4.1 模式 A：创作优先（默认）

```
┌─┬────────────────────────────────────┬─────────────┐
│ │  最终成品（70%）                    │  辩论摘要    │
│项│  ┌──────────────────────────────┐ │  （30%）     │
│目│  │                              │ │             │
│树│  │  正文阅读区                   │ │  ●大纲       │
│ │  │  （字体大、行距宽）            │ │  ●初稿       │
│ │  │                              │ │  ●审稿       │
│ │  └──────────────────────────────┘ │  ●终审       │
│ │  ──────────────                   │             │
│ │  本轮指令（折叠）                  │  [展开看辩论]│
│ │  [开始创作] [继续下一轮]            │             │
└─┴────────────────────────────────────┴─────────────┘
```

- 正文阅读区**占主视野**，仿 Notion 文档阅读体验
- 本轮指令默认折叠，点击展开
- 辩论摘要只显示步骤状态，点击展开完整辩论

### 4.2 模式 B：辩论优先

```
┌─┬─────────────────────┬─────────────────────────────┐
│ │  本轮指令 + 操作      │  辩论流（完整展示）          │
│项│  ┌─────────────────┐│  ●主编 → 大纲                │
│目│  │场景概述          ││  ●作家 → 初稿               │
│树│  │特殊指令          ││  ●批评家 → 审稿（红色高亮）  │
│ │  │目标字数          ││  ●一致性 → 校对              │
│ │  │[开始创作]        ││  ●主编 → 终审                │
│ │  └─────────────────┘│  ●润色师 → 润色              │
│ │  ──────────────     │  ──────────────              │
│ │  最终成品（小窗）     │  本轮重大决策                │
│ │  [展开看正文]        │  本轮 token：1,234            │
└─┴─────────────────────┴─────────────────────────────┘
```

- 辩论流占大头，类似 ChatGPT 多角色对话
- 适合调试 prompt、观察 AI 协作

### 4.3 切换方式

- TopBar 中间一个 toggle（图标+文字）
- 快捷键：`Cmd/Ctrl + Shift + M`
- 切换时有 200ms 滑动过渡动画

## 六、关键交互

### 6.1 快捷键体系

| 快捷键 | 功能 |
|--------|------|
| `Cmd/Ctrl + K` | 命令面板 |
| `Cmd/Ctrl + N` | 新建项目 |
| `Cmd/Ctrl + S` | 保存设定 |
| `Cmd/Ctrl + Enter` | 开始/继续创作 |
| `Cmd/Ctrl + 1/2/3` | 切换 Tab（设定/创作/历史） |
| `Cmd/Ctrl + Shift + M` | 切换模式（创作/辩论） |
| `Cmd/Ctrl + Shift + L` | 切换深浅主题 |
| `Cmd/Ctrl + B` | 切换左侧栏 |
| `Cmd/Ctrl + /` | 切换右侧抽屉 |
| `Esc` | 关闭弹窗/工具栏 |

底部 StatusBar 显示当前上下文相关的快捷键提示。

### 6.2 画中画（PiP）

辩论流可以"画中画化"：

- 点击辩论流标题栏的 📌 图标
- 辩论流变成右下角浮窗（280×400px）
- 浮窗可拖动到任意位置（位置记忆）
- 浮窗可最小化为图标
- 用户切到设定 Tab 时也能看到创作进度

### 6.3 拖拽

- **面板边界**：拖动调宽
- **角色卡**：拖动排序
- **章节树节点**：拖动重排序（未来支持）
- **历史轮次**：拖到"回收站"图标移入回收站

### 6.4 后台运行

- 创作过程在后端是异步的
- 前端切换 Tab/项目时，创作不中断
- 右下角始终有"运行中"小图标（点击回到创作）
- 创作完成时弹通知，即使在其他 Tab

### 6.5 状态持久化

存到 localStorage：
- 主题选择
- 模式选择
- 面板宽度
- 左侧栏折叠状态
- 上次打开的项目
- PiP 浮窗位置

## 七、辅助细节

### 7.1 加载状态

- **骨架屏**：项目列表、历史轮次加载时显示灰色占位条
- **流式打字**：作家输出时显示打字光标动画
- **环形 spinner**：短任务（< 2s）显示
- **进度条**：长任务（创作流程）显示步骤进度

### 7.2 错误处理

- 错误用红色 toast 弹出，不自动消失
- 错误信息人话翻译（已实现）
- 严重错误（无法连接后端）显示满屏遮罩 + 重连按钮
- 单个委员失败时不阻塞流程，红色高亮标记

### 7.3 空状态

- 无项目时：大图标 + "创建你的第一本书"按钮
- 无轮次时：温和的提示 + 引导填写场景概述
- 无角色时：提示先去设定 Tab 添加角色

### 7.4 移动端适配

MVP 阶段**不做移动端**。预留：
- 所有布局用 flex/grid 而不是固定像素
- 触摸事件兼容（如长按代替右键）
- 关键断点：768px 以下隐藏左侧栏，强制折叠右侧抽屉
